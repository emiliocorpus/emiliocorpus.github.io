BLOG #6:Enumerable Methods, #map
SUNDAY, 5/24/2015

If you are like me, someone relatively new to Ruby (yet alone the coding world), there is a lot to take in. Having recently started DevBootcamp, I have been presented many challenges and exercises regarding strings, arrays and most recently hashes. Over the last few weeks I have gotten more familiar with these types of objects, and I now have a steady foundation of knowledge on collections. However, any foundation can be fortified, and today we will go through Enumerables and specifically look at one method called #map.

Enumerable is a module. It offers a large collection of methods that are quite helpful in handling data collections, like Hash and Array. Say you have the following:

array_of_test_scores = [45,32,15,19,21,45,42,35,34,12]

Like the variable name states, this is array of test scores. Let's say the test scores are out of 50, and each score in the array represents a student. The teacher is feeling pretty friendly today and wants to give all her students an extra point on the test. How can we change the scores without having to redefine the array?

If you guessed the Enumerable#map, then give yourself a pat on the back! So here is what the code would look like if we wanted to add 1 to everyone's score.

array_of_test_scores.map{|x|x+1}
=> [46,33,16,20,22,46,43,36,35,13]

As you can see, we call the method .map with a code block within the following {}. What .map does is it goes through each item in the array, and executes the code that you give it. The |x| in the block represents each item, and the following function x + 1 is executed for each item. This tranforms our array! But if we were to call the array and print or p it, we would see our original inputs, that is the scores before adding 1.

Why is this? Well, this is because we called the non-destructive version of .map, which actually preserves our original array. If we were to make it destructive, that is transform/change the original array, it is simple, we add a ! to the end of the call like this:

array_of_test_scores.map!{|x|x+1}
=> [46,33,16,20,22,46,43,36,35,13]

So when we print or p like this:

p array_of_test_scores

The output on the console would show:

=> [46,33,,20,22,46,43,36,35,13]

our new scores!

This method makes our lives so much easier in terms of manipulating different data collections! This is all thanks to the module Enumerable! Here is a link to Enumerables. There you will find all other Enumerable methods. I recommend looking at the the .each method, which is actually quite integral in the defining proceses of all other Enumerable methods. .each is what made .map go through every item in the object!
