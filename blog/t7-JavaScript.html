BLOG 12:Javascript Vs. Ruby
MONDAY, 6/15/2015

I am still in the beginning of my programming voyage and have been mostly absorbing Ruby over the last 7 weeks. However this week I have become more familiar with a new languange, Javascript. I had some experience with Javascript through codecademy several months ago as well as Eloquent Javscript. These are great introductory resources I recommend to any novice like myself. Javascript has many differences yet at the same time has so many parallels with Ruby. Today I will be discussing some of this differences/similariites.

From what I have learned so far, Ruby has a larger bag of tricks in comparison to Javascript. I discovered this the hard way when attempting to write Javascript code. I found that javascript does not have have many of the enumerable methods, such as each, map and select, that Ruby has built into its libraries. But like Ruby, one can create these functions and then incorporate into their code. Let's start small and work our way up to creating one of these functions.

First of all, in Ruby one can define a variable like the following:

RUBY

my_name = "Emilio"
puts my_name
# => Emilio


In Javascript defining a variable is different but pretty simple. When you are defining a variable for the first time you would use the following:

JAVASCRIPT

var my_name = "Emilio";
console.log(my_name);
// => Emilio
my_name = "the many faced god";
console.log(my_name);
// => the many faced god

So when defining a variable for the first time you have to write var in front of the variable you are naming. Like ruby, you use an equals sign to set that to the value desired, in this case it was first my name, "Emilio". To print something to the console you must use console.log(), and the thing you wanted to be printed goes within the parens. This is similar to what puts does in Ruby. Once you have a variable assigned to a value, you can reassign that variable to a different value, but note that you do not need to write the var again. Also, in Javascript it is important to end each expression with a ;. In Ruby you did not need such separators, but in Javascript a missing ; can destroy your whole code!

In Ruby one can adjust the indices in Strings AND Arrays. However in Javascript you can only adjust arrays, as Strings are immutable once they are defined. Javascript does not have any other collection based datatypes, unlike Ruby which has the Hash collection. However, Javascript does seem to have something similar to Hashes, and these are called objects. Let's define an object below and see if you understand the test code I write after defining:

So in comparison to Ruby, Javascript objects are kind of like Hashes in that can define different properties to different values. If you try to call a propety that does not exist, you will get undefined in return. However, like in Ruby, you can create properties outside of the explicit Object definition( objectName.propertyName = valueDesiredHere). I did this when I defined the favoritSport property to equal "basketball".

Javascript also has the ability to use loops like Ruby does. The syntax for the infamous while loop is the following:

JAVSCRIPT

while (condition here within parens) {

   code block here

}

Its use is similar to that of Ruby, in that the code block will be executed as long as the condition is true. Another type of loop very integral to Javascript is the for loop. An example is described below

JAVASCRIPT

var array = [1,2,3,4,5];
for (var i = 0; i < array.length; i++) {
   console.log(array[i]);
}
// => 1
// => 2
// => 3
// => 4
// = >5


In this above code block we have an array defined with 5 different values (the numbers 1-5). Inside the parens of the for loops you will notice we have 3 different expression (all separated withe the ;). The first expression is setting a variable equal to zero. The second expression is basically a while loop without the while statement. You will notice the .length property that is called on the variable array. This does exactly what you would expect it to do in Ruby, it evaluates to the length (ie. number of items in the array). The third expression changes the value of i to and increments it by 1. So as long as the value of the variable i is less than the number of items in the array, the code block will be execute. In this case we are printing the value at the i index of the array until the variable i is no longer less than the length of the array. Although the format of the for loop is much different than Ruby's while loop, the ideas/fundamentals behind is the same: while a certain condition is true execute the code and stop when the condition is no longer true.

If you remember in the beginning of this blog I told you that Javascript does not have a built-in .each method like Ruby does. Let's create it now:

JAVASCRIPT

var each = function(array, func) {
   for (var i=0; i < array.length; i++) {
      func(array[i]);
   }
};


You are probably commenting right now, "what the hell did he just do...". Well no worries friend, I am here to explain! Javascript has the ability to create functions. This is similar to writing methods in Ruby. The function we created here has two parameters: an array and another argument func (which is a function). Insided our codeblock of the each function is a for loop (just like the previous for loop), and within the for loop's code block is a segment of code that executes the function you gave it through the each function. So basically what it will do is execute the function you give it for each item in the array! So say we wanted to print out a list of the items, 3 times per item, we would do the following:

JAVASCRIPT

var each = function(array, func) {
   for (var i=0; i < array.length; i++) {
