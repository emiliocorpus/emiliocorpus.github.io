BLOG 14: An Intro to Regular Expressions
SUNDAY, 6/21/2015

As defined by the ruby-doc.org resource site, regular expressions are patterns which describe the contents of a string. It is a class/object provided to many different programming languages. It allows a programmer to change certain values of strings or find values based on if the string has those specified patterns. In this tutorial we will use Ruby to illustrate these concepts.

A regular expression pattern can be composed of simple characters. It can look like /abc123/ or /Emilio is a cool guy/. In Ruby, there is a Regexp class. You can create a new Regexp by using the /reg exp in here/ and %r{reg exp in here}, or the Regexp.new literal constructor.

Let us look at an example

test_string = "Steph Curry is the chef"

if test_string =~ /chef/
   p "Oh, you know it"
else
   p "This should not print"
end
# => "Oh, you know it"

In Ruby, to check if a string or variable containing a string has a certain regular expression you must use the =~, (although some built in methods already have this integrated into its code block).

Except for control characters, which include:
      (+?.*^$()[]{}|\)
all characters match themselves. You can escape a control character by preceding it with a backslash.

test_string2 ="(+?>"

if test_string2 =~ /\+/
   p "Bingo!"
else
   p "This should not print"
end
# => "Bingo!"

This is a link that has tables of regular expressions pattern syntax for Ruby. In this tutorial we will be going through some but not all of these points. I would definitely recommend using www.tutorialspoint.com as they have a great overview of Regular Expressions in Ruby.

The first pattern we will look at is the [..] pattern, which matches any single character in the brackets. There is also a [^..], which matches any single character that is not in the brackets following the ^.

test_string3 = "Harry plays the xylophone"

if test_string3 =~ /[bzqxfz]/
   p true
else
   p "This will not print"
end
#=> true

Since the pattern /[bzqxfz]/ has an 'x' in it, the first if statement will be executed.

The \s pattern represents a space character.

test_string4 = "Thissentencehasnospaces"

if test_string4 =~ /\s/
   p "Since the sentence has no spaces"
   p " this will not print"
else
   p "But this will!!"
end
# => "But this will!!"



One important and highly used method in Ruby is the gsub method. This returns a copy of the str with all occurences of the pattern substituted with a new pattern that you enter as a second argument. It looks like so:

test_string5 = "My password is ****"

test_string5.gsub!(/\*\*\*\*/, "1234")

p test_string5

# => "My password is 1234"


Please note that I used the ! symbol in order to change the original array. You can use gsub without it. There is also a sub method that only changes the first pattern it comes across. It too has a ! destructive property.

Lastly, we are going to look at an Enumerable method called grep. This method returns an array of every element in the enumerable object that matches or has the regular expression pattern. Lets say we have the following array:

test_array1 = ["Gabriel", "Emilio", "Alex", "Bob", "Gerald", "George", "Igor","Zoolander"]

Let's try and select every item that has a "G" or "Z" in it.


p test_array1.grep(/[GZ]/)

# => ["Gabriel", "Gerald", "George", "Zoolander"]




If you noticed, the name "Igor" did not get selected even though it has a "g" in its string. Regular expressions take case into account. If we had used /[GgZ]/ then "Igor" would have been in the returned array. This method makes looking/checking for certain string values much easier.
